# Confiq

Универсальный менеджер конфигураций с гибкой системой шаблонов.

> **📝 Примечание:** Confiq — это инструмент для управления конфигурациями, который может быть использован с различными API. В данном репозитории в качестве примера реализована интеграция с Cloudflare API для работы с WARP конфигурациями.

## ⚠️ Важная информация

**Автор проекта не связан с компанией Cloudflare.** Использование публичного API Cloudflare для создания и управления конфигурациями WARP представлено исключительно в **ознакомительных и образовательных целях**.

## 🚀 Основные возможности

### Для пользователей
- ✅ Создание конфигураций различных типов (WireGuard, AmneziaWG, Clash)
- ✅ Просмотр конфигураций с QR-кодами
- ✅ Скачивание конфигураций в форматах `.conf` и `.yaml`
- ✅ Управление своими конфигурациями
- ✅ Ограничения на количество конфигов на пользователя

### Для администраторов
- ✅ Управление пользователями
- ✅ Управление endpoints для подключения
- ✅ Управление типами конфигураций с шаблонами Jinja2
- ✅ Настройка поддерживаемых типов для каждого endpoint
- ✅ Просмотр всех конфигураций в системе
- ✅ Массовые операции с конфигурациями

## 📦 Быстрый запуск

```bash
# 1. Клонирование репозитория
git clone https://github.com/bekirovtimur/confiq.git
cd confiq

# 2. Настройка .env файла
cp .env.example .env
# Отредактируйте .env, укажите CERTBOT_DOMAIN, CERTBOT_EMAIL, ADMIN_PASSWORD, SECRET_KEY

# 3. Запуск
docker compose up -d
```

Приложение будет доступно по HTTPS через несколько минут.

**Подробная инструкция:** [`docs/DEPLOYMENT.md`](docs/DEPLOYMENT.md)

## 📚 Использование

### Для администратора

1. Войдите в систему (`/auth/login`)
2. Создайте типы конфигураций (Панель управления → Типы конфигураций)
3. Создайте endpoints (Панель управления → Endpoints)
4. Создавайте пользователей (Панель управления → Пользователи)

### Для пользователя

1. Войдите в систему
2. Создайте конфигурацию: выберите endpoint и тип
3. Скачайте файл конфигурации или отсканируйте QR-код

## 🔌 API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET/POST | `/auth/login` | Вход в систему |
| GET | `/auth/logout` | Выход из системы |
| GET | `/user/dashboard` | Панель пользователя |
| GET/POST | `/user/configs/create` | Создание конфигурации |
| GET | `/admin/dashboard` | Панель администратора |
| GET | `/admin/config-types` | Типы конфигураций |

- **База данных**: SQLite

## 👥 Группы пользователей

В системе реализована поддержка **групп пользователей** для управления доступом к endpoints.

### Как работают группы

- Каждый пользователь может быть назначен в группу (или оставаться без группы)
- Группы определяют, к каким endpoints имеют доступ пользователи
- Администратор может привязать endpoints к группам
- Пользователи видят и могут создавать конфигурации только на доступных им endpoints

### Управление группами

1. Создайте группы в панели администратора (Панель управления → Группы)
2. Назначьте пользователей в группы при создании/редактировании пользователя
3. Привяжите endpoints к группам при создании/редактировании endpoint

## 🔄 Миграция с предыдущих версий (до появления групп)

⚠️ **ВАЖНО: Перед миграцией обязательно сделайте резервную копию базы данных!**

```bash
# Создание бэкапа
cp data/warp_manager.db data/warp_manager.db.backup.$(date +%Y%m%d_%H%M%S)
```

Если вы используете старую версию без поддержки групп, выполните миграцию:

```bash
# Запуск миграции
python3 migrations/migrate_add_groups.py data/warp_manager.db
```

Миграция создаст:
- Таблицу `groups` для хранения групп
- Колонку `group_id` в таблице `users`
- Таблицу связи `group_endpoints` для привязки endpoints к группам
- Таблицу связи `endpoint_config_types` (если отсутствует)

После миграции все существующие пользователи будут без группы (доступ ко всем endpoints).

## 📝 Информационный баннер

На страницах пользователей отображается информационный баннер, который можно использовать для размещения важных объявлений, информации о поддержке проекта или других сообщений.

### Где отображается баннер

Баннер отображается на следующих страницах:
- **Панель пользователя** (`/user/dashboard`) — над блоком "Быстрые действия"
- **Список конфигураций** (`/user/configs`) — под статистикой, над списком конфигураций
- **Просмотр конфигурации** (`/user/configs/{id}`) — под QR-кодом

### Редактирование содержимого баннера

Содержимое баннера хранится в файле [`info.html`](info.html) в корне проекта. Файл поддерживает HTML-разметку, что позволяет:
- Форматировать текст (жирный, курсив, заголовки)
- Добавлять ссылки
- Создавать списки
- Использовать стили

### Включение и отключение баннера

Отображение баннера управляется переменной окружения `INFO_BANNER` в файле `.env`:

```bash
# Включить баннер (значение по умолчанию)
INFO_BANNER=true

# Отключить баннер
INFO_BANNER=false
```

После изменения переменной окружения необходимо **перезапустить контейнеры**:

```bash
docker compose restart
```
